---------------------------DataBase Schema-----------------------------
Use Supabase for the Backned 
1. ROLES

Stores all available roles for company employees.

Column	Type	Description
role_key (PK)	VARCHAR	Unique role identifier (e.g., R001)
role_name	VARCHAR	Role name (Super Admin, HR Manager, Hiring Manager, Viewer)
description	TEXT	Brief description of the role
is_active	BOOLEAN	Indicates whether the role is active
2. ADMIN

Stores all company employees who can access the admin portal.

Column	Type	Description
admin_id (PK)	INT	Unique admin ID
full_name	VARCHAR	Employee name
email	VARCHAR	Employee email
password	VARCHAR	Encrypted password
role_key (FK)	VARCHAR	References ROLES.role_key
department	VARCHAR	Department
phone	VARCHAR	Contact number
account_status	ENUM	Active / Inactive
created_at	TIMESTAMP	Account creation time
updated_at	TIMESTAMP	Last update time
3. JOBS

Stores all job postings.

Column	Type	Description
job_id (PK)	INT	Unique job ID
job_title	VARCHAR	Job title
job_description	TEXT	Job description
expected_ctc	VARCHAR	Salary package
job_location	VARCHAR	Location
employment_type	VARCHAR	Full-Time / Internship
openings	INT	Number of openings
application_start_date	DATE	Start date
application_end_date	DATE	End date
job_status	ENUM	Open / Closed / Shortlisting Closed
resume_cutoff_score	INT	Minimum AI resume score
evaluation_prompt	TEXT	Prompt used for AI evaluation
email_template	TEXT	Email template
created_by (FK)	INT	References ADMIN.admin_id
created_at	TIMESTAMP	Creation time
updated_at	TIMESTAMP	Last update
4. STUDENTS

Stores all candidates who apply for jobs.

Column	Type	Description
student_id (PK)	INT	Unique student ID
full_name	VARCHAR	Candidate name
email	VARCHAR	Candidate email
phone	VARCHAR	Contact number
job_id (FK)	INT	Applied job
resume_url	TEXT	Resume file URL
parsed_resume_json	JSON	Extracted resume details
resume_score	DECIMAL	AI resume score
application_status	ENUM	Applied / Shortlisted / Rejected / Hired
created_at	TIMESTAMP	Application time
5. SHORTLISTED_STUDENTS

Stores candidates who passed resume screening.

Column	Type	Description
shortlisted_id (PK)	INT	Unique ID
student_id (FK)	INT	References STUDENTS.student_id
video_url	TEXT	Uploaded introduction video
video_score	DECIMAL	AI video evaluation score
aptitude_score	DECIMAL	Aptitude test score
final_score	DECIMAL	Overall score
recommendation	ENUM	Hire / Consider / Reject
current_stage	VARCHAR	Video / Aptitude / Final Review / Completed
updated_at	TIMESTAMP	Last evaluation update
Relationships
ROLES (1) → ADMIN (M)
ADMIN (1) → JOBS (M)
JOBS (1) → STUDENTS (M)
STUDENTS (1) → SHORTLISTED_STUDENTS (0..1)
